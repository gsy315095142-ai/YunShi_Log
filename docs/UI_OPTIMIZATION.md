# 运势 Log — UI 排版优化建议

> 文档版本：v1.0  
> 创建日期：2026-07-20  
> 状态：建议已评审通过，第 1～7 项已实施

本文档记录对前端界面排版的整体评审结论与优化建议，作为后续 UI 迭代的依据。

---

## 一、全局问题

### 1. 样式重复、缺少设计变量 ✅ 已实施

**问题**：`.card` 样式在 `DailyPage.css`、`ProfilePage.css`、`AIPage.css` 中各复制一份；主题色（`#7c3aed`、`#8b7faa` 等）散落各处，改主题需多处修改。

**建议**：在 `index.css` 定义 CSS 变量（`--color-primary`、`--color-text-secondary`、`--radius-card` 等），`.card` 抽为全局公共类，各页面只写差异化样式。

### 2. 主内容区缺少页面标题 ✅ 已实施

**问题**：顶栏只有「运势Log」，用户只能靠底部 Tab 高亮判断当前所在页面。

**建议**：顶栏增加当前页面标题（个人信息 / 每日记录 / AI 测算），随路由自动切换。

### 3. 触控目标偏小 ✅ 已实施

**问题**：月份切换按钮 36px、记录操作按钮约 30px 高，低于移动端 44px 的建议触控尺寸。

**建议**：主要可点区域高度提升至 40px 以上。

### 4. 内页输入框缺少 focus 态 ✅ 已实施

**问题**：登录页输入框有 focus 光环，内页（记录、个人资料）输入框没有，体验不一致。

**建议**：在 `index.css` 为 `input / textarea / select` 增加全局 focus 样式。

---

## 二、每日记录页（默认首页，问题最集中）

### 5. 日历格子过挤 ✅ 已实施

**问题**：64px 高的格子里塞日期 + 0.65rem 两行摘要，手机上难以阅读；后端返回的 `count`（记录条数）字段前端未使用。

**建议**：摘要精简为一行省略号；增加记录条数角标；格子保持紧凑但信息层级更清楚。

### 6. 选中日期后需向下滚动 ✅ 已实施

**问题**：记录面板直接接在日历下方，手机上点完日期看不到反馈，需要滚动才能看到记录列表。

**建议**：改为**底部弹出层（bottom sheet）**，点日期即从底部滑出当日记录面板，符合移动端交互习惯，附半透明遮罩与关闭按钮。

### 7. 编辑状态无法取消 ✅ 已实施

**问题**：进入编辑态后没有「取消编辑」按钮，只能靠切换日期退出。

**建议**：编辑时出现「取消编辑」按钮，恢复为新增模式。

### 8. 周末样式缺失 ✅ 已实施

**问题**：TSX 中为周末格子加了 `.weekend` class，但 CSS 中从未定义对应样式（死代码）。

**建议**：补上周末日期着色（柔和的暖红色数字），周六日一眼可辨。

### 9. 「今天」标记与摘要重叠 ✅ 已实施

**问题**：今天的小圆点 `::after` 定位在格子底部居中，有记录时会和摘要文字重叠。

**建议**：将今天标记移至格子右上角，与摘要区域错开。

### 10. 删除使用浏览器原生 confirm ✅ 已实施

**问题**：原生 `confirm()` 弹窗风格与整体 UI 突兀。

**建议**：改为两段式删除——点击「删除」后按钮变为「确认删除」，再次点击才执行，误触成本低。

## 三、AI 测算页

### 11. 聊天区嵌套滚动 ✅ 已实施

**问题**：聊天窗口限高 `50vh`，藏在可滚动的 `app-main` 里，手机上双层滚动手感差；`min-height: 420px` 与 `max-height: 50vh` 约束互相冲突。

**建议**：聊天页改为全高 flex 布局——配置卡片固定顶部，消息区自动撑满剩余空间并内部滚动，输入框固定在底部 Tab 栏上方（聊天类应用标准做法）。

---

## 四、个人信息页

### 12. 命理信息展示单调 ✅ 已实施

**问题**：「命理信息」为纯文本表格，作为命理类应用的卖点信息，视觉权重不足；出生时间在表单和只读卡中重复展示。

**建议**：改为 2×2 视觉瓷贴（星座 / 生肖 / 五行 / 农历），延续登录页五行瓷贴的视觉语言；出生时间从只读卡中移除（表单已有）。
